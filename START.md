# LD LATTE Blogger AI Agent

Проект подбирает fashion-блогеров для бартерного сотрудничества с LD LATTE — компанией женской одежды на Wildberries и Ozon. Он анализирует эталонную базу, строит портрет подходящего автора, ищет и обогащает публичные профили, ранжирует кандидатов и готовит персонализированные черновики офферов. В проекте используются Python, Pydantic, OpenAI Responses API, Google Sheets, Tavily, Apify, CSV и Markdown. Модульная архитектура разделяет источники данных, AI-анализ, поиск, enrichment, scoring и отчётность, чтобы каждый этап можно было независимо тестировать, заменять и запускать в mock- или dry-run-режиме.

## ЧАСТЬ 2. Автоматизация подбора трендовых товаров

Я бы автоматизировал подбор трендовых товаров.

Сегодня маркетологу приходится регулярно анализировать маркетплейсы, социальные сети, поисковые запросы и продажи, чтобы понимать, какие модели одежды начинают набирать популярность. Это занимает много времени и часто зависит от субъективной оценки.

Я бы построил ИИ-сервис, который ежедневно собирает информацию из различных источников, анализирует изменения спроса, упоминания в социальных сетях, частоту появления определённых моделей, цветов и фасонов, после чего формирует список товаров, которые имеют наибольший потенциал роста.

Для реализации понадобятся:

• данные о продажах компании;

• статистика маркетплейсов;

• поисковые тренды;

• данные социальных сетей;

• LLM для анализа и объединения информации.

Работу решения можно проверить, сравнив прогноз системы с реальными продажами через несколько недель.

Через месяц после внедрения основной метрикой эффективности станет доля товаров, рекомендованных системой, которые действительно показали рост продаж. Дополнительно можно оценивать сокращение времени, которое маркетологи тратят на поиск новых трендов.

# Известные ограничения

• Apify используется как временный источник enrichment Instagram-профилей.

• В production рекомендуется заменить Apify собственной системой сбора данных или официальным источником.

• Tavily используется только как источник поисковых гипотез.

• Все найденные кандидаты проходят ручную проверку.

• OpenAI не принимает финальное решение.

• Система является инструментом поддержки принятия решений, а не полностью автономным агентом.

# Как запустить

## Установка

```bash
python3.12 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
cp .env.example .env
```

## Dry Run

Показывает план реального поиска и лимиты расходов, не вызывая Tavily, Apify или OpenAI:

```bash
python -m src.main --find-real-bloggers --dry-run-final-v2
```

## Полный запуск

Полностью офлайн запускает демонстрационный mock-пайплайн и сохраняет CSV-результат:

```bash
python -m src.main
```
