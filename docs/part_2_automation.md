# Часть 2. AI-анализ отзывов на Wildberries и Ozon

## Процесс для автоматизации

Для LD LATTE полезно автоматизировать первичный разбор новых отзывов о женской
одежде на Wildberries и Ozon. Цель — быстрее замечать повторяющиеся проблемы с
посадкой, размерами и качеством, готовить единообразные черновики ответов и не
пропускать критичные обращения. Это проектное предложение, а не заявление об
опыте работы с внутренними системами маркетплейсов.

## Предлагаемый сценарий

1. По расписанию получать новые отзывы из разрешённой выгрузки, официального API
   или кабинета продавца. Для каждого отзыва сохранять площадку, товар, артикул,
   размер, рейтинг, текст и дату; персональные данные не дублировать без нужды.
2. Исключать уже обработанные записи по стабильному идентификатору и проверять
   обязательные поля.
3. Классифицировать отзыв по темам: размерная сетка, посадка, ткань, пошив,
   цвет, комплектация, упаковка, доставка, возврат и другое. Отдельно определять
   тональность и срочность.
4. Группировать повторяющиеся жалобы по товару и размеру. Например, рост доли
   сообщений «маломерит» для конкретного артикула и размера должен стать
   сигналом для проверки карточки и размерной сетки.
5. Генерировать черновик короткого ответа только на основании отзыва и
   утверждённой базы правил бренда. Не обещать компенсации, возврат или свойства
   товара, которых нет во входных данных.
6. Передавать ответственному сотруднику критичные отзывы: возможный брак,
   безопасность, массовая повторяемость одной проблемы, рейтинг 1–2 или высокая
   негативная тональность. Остальные черновики также публиковать только после
   ручного подтверждения.

## Инструменты и данные

- официальный API/разрешённая CSV-выгрузка Wildberries и Ozon;
- Python, pandas и Pydantic для валидации и агрегации;
- LLM со Structured Outputs для тем, тональности и черновика ответа;
- PostgreSQL или Google Sheets для небольшого пилота и истории обработки;
- Telegram или корпоративный мессенджер для уведомлений ответственному;
- планировщик cron, GitHub Actions либо оркестратор компании.

Минимальная запись аудита: идентификатор отзыва, версия промпта и модели,
классы, уверенность, черновик, финальный ответ, редактор и время подтверждения.

## Проверка качества

До запуска собрать размеченную выборку отзывов и согласовать категории с
менеджером. На пилоте сравнить AI-классификацию с ручной разметкой, отдельно
проверить критичные классы и ложные срабатывания. Черновики оценивать по фактам,
тону бренда и отсутствию неподтверждённых обещаний. Низкая уверенность,
непонятный контекст и новые темы всегда направляются человеку. Раз в неделю
просматривать ошибки и обновлять справочник правил; автоматическую публикацию
не включать до устойчивого качества.

## Метрики через месяц

- среднее время ответа на отзыв;
- доля отзывов, обработанных в установленный SLA;
- доля AI-ответов, отредактированных сотрудником перед публикацией;
- повторяемость жалоб по артикулам и размерам;
- изменение рейтинга карточек товаров;
- экономия рабочего времени команды;
- дополнительно: полнота выявления критичных отзывов и точность тематической
  классификации на контрольной выборке.
